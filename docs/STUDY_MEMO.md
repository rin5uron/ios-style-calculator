### 【2025/05/18】  
#### テーマ：`rem` と `vw` の使い分け（電卓アプリの実装例から）

**学んだこと**
- `rem` はルート（`html`）のフォントサイズを基準とする相対単位
  - 通常、1rem = 16px（ブラウザ標準）
  - サイト全体で統一感あるフォント設計に最適
  - ユーザーのブラウザ設定（拡大など）にも対応できるため、**PC向けに特に有効**

- `vw` は「Viewport Width（画面幅）」の1%を表す単位
  - 1vw = 画面幅の1%
  - **スマートフォンなどの小さい画面で自動的に文字サイズが調整される**
  - 表示領域が狭いスマホでも、読みやすくかつ美しく表示できる

- `rem` は全体設計の基本、`vw` は実機での見た目重視のチューニングに使い分けるのが効果的

**活用例**
- 電卓アプリのボタン内テキストサイズを、スマホでは `vw` にして調整：

```css
/* PC・全体用 */
.number-button {
  font-size: 1.75rem;
}

/* スマホ用（幅400px以下） */
@media (max-width: 400px) {
  .number-button {
    font-size: 8vw;
  }
}
```

### 【2025/05/18】
#### テーマ：`display: flex; align-items: ; justify-content: ;` の3点セットの役割と電卓ディスプレイでの使い方

**学んだこと**
- `display: flex;`：子要素を柔軟に並べられるようになる（横・縦・中央揃えなどに対応）
- `justify-content:`：**横方向**の整列方法を指定（`flex-end` で右寄せ）
- `align-items:`：**縦方向**の整列方法を指定（`flex-end` で下寄せ）
- `flex-end` 自体に「右下」の意味はない → 並びの終点（最後）に寄せるという意味
- この3点を組み合わせると、子要素が親要素の **右下にピッタリ配置** される

**活用例**
- iPhone電卓のように、表示される数字を右下にピッタリ配置したいときに最適
- CSSで以下のように指定すれば、表示エリア内で数値が美しく右下に収まる

```css
#display {
  display: flex;
  justify-content: flex-end;  /* 横方向 → 右寄せ */
  align-items: flex-end;      /* 縦方向 → 下寄せ */
  font-size: 4rem;
  height: 100px;
  padding: 10px 20px;
}
```

### 【2025/05/18】
#### テーマ：`rem` / `em` / `px` の使い分け

**学んだこと**
- `rem`：ルート要素（`html`）のフォントサイズを基準にした相対単位（通常16px）
- `em`：親要素のフォントサイズを基準にした相対単位（入れ子に注意）
- `px`：絶対単位。デバイスに依存しない固定サイズ
- HTMLのデフォルトのフォントサイズは **16px**（ブラウザ標準）

**活用例**
| 単位 | 意味 | 基準 | 特徴・用途 | よく使う場面 |
|------|------|------|------------|--------------|
| `px` | ピクセル | 固定値 | 画面サイズに依存せず正確に指定できる | ボーダー、余白、精密な調整に最適 |
| `rem` | ルート em | `html`のフォントサイズ（通常16px） | ページ全体で統一感を出せる | 見出し・基本フォントサイズなど |
| `em` | 相対 em | 親要素のフォントサイズ | コンポーネント単位で調整しやすい | ボタン内テキスト、パーツのサイズ |
| `%` | パーセント | 親要素のサイズ | レイアウトでよく使う | 幅や高さの可変レイアウト |
| `vw` | Viewport Width | 画面の幅の1% | デバイス幅に対して自動調整 | スマホUIの文字・ボタンサイズ |
| `vh` | Viewport Height | 画面の高さの1% | 全画面表示などで便利 | Heroセクションや縦中央配置 |
| `ch` | Character width | "0"の文字幅 | テキスト入力の横幅に便利 | `input` の長さ指定など |

---

**使い分けのコツ**
- 基本文字サイズには `rem`
- パーツ調整には `em`
- レイアウトには `%`
- スマホ表示のボタンやフォントには `vw/vh`

### 【2025/05/15】
#### テーマ：Markdownファイル名の大文字と小文字の使い分けについて

**学んだこと**
- 
Markdownファイル名において、大文字と小文字には技術的な差はないが、視認性や分類の目的で使い分けがされる。

---

大文字を使う理由

#1. 視認性の向上
- 大文字のファイル名はファイルツリーで目立つ。
- 文書ファイルをすぐに識別できる。

### 2. 役割の明示
以下のように用途別に区別されることが多い：

| ファイル名         | 用途                     |
|------------------|--------------------------|
| README.md        | リポジトリの概要・導入文書   |
| PROCESS_LOG.md   | 開発や学習の進捗記録        |
| TECH_SPEC.md     | 技術仕様書                 |
| TODO.md          | タスクリスト               |


---

## 推奨ルール（例）

- 文書系 `.md` ファイル：**大文字**で始める
- コードファイル：**小文字**で統一

※チームや個人でルールを明示しておくと混乱を避けやすい

**活用例**
- コード：小文字（例：`main.js`, `index.html`）
- 文書：大文字（例：`README.md`, `LOG.md`）

### 【2025/05/15】
#### テーマ：San FranciscoフォントをCSSで使いたいとき

**学んだこと**
- ## ❓ San Franciscoってなに？(2025/05/15)

- Appleが作った**iPhoneやMacの標準フォント**
- iPhoneの時計アプリ・メッセージアプリなど、AppleのほぼすべてのUIで使われている
- スタイリッシュで読みやすく、Appleらしさが出る！

---

## ❓ CSSでこう書きたくなるけど…

```css
font-family: 'San Francisco', sans-serif;
```

⬆️ これは **使えない！動かない！** 😢

---

## ❗ なぜ `'San Francisco'` と書いても動かないの？

- **Appleはこのフォントに"名前"をつけていない**（びっくりだけど事実）
- インストールされてるフォントじゃなくて、**システムの中に隠れてる特別なフォント**
- WindowsやAndroidには当然入ってない
- Appleの中でも「San Francisco」という**名前では指定できないようにしてある**

---

## ✅ じゃあどうすればいいの？

Appleが用意してくれてる"魔法のキーワード"を使う！

```css
font-family: -apple-system;
```

これを書くだけで：

- iPhone → San Francisco になる✨
- Mac → San Francisco になる✨
- WindowsやAndroidでは → 近いフォントが代わりに使われる

---

## ✅ 実際によく使われてるおすすめの書き方

```css
font-family: -apple-system, BlinkMacSystemFont, "Helvetica Neue", sans-serif;
```

| フォント名              | 説明                                        |
|-------------------------|---------------------------------------------|
| `-apple-system`         | Apple公式のSan Francisco指定用キーワード   |
| `BlinkMacSystemFont`    | Chrome/macOSで使われるSan Francisco対応     |
| `"Helvetica Neue"`      | 旧iOSで使われてたスタイリッシュな代替文字   |
| `sans-serif`            | 最後の保険（どの環境でもゴシック体になる） |

---

## ✨ 覚えておくべきまとめ

- `'San Francisco'` はCSSで**書いても効かない**
- Apple風にしたいときは **`-apple-system` を使おう**
- 複数フォントを並べることで、**いろんな環境でも見た目が安定する**

---

## 💡 具体例（CSSで使うとき）

```css
body {
  font-family: -apple-system, BlinkMacSystemFont, "Helvetica Neue", sans-serif;
}
```
